# Документация API

Полное описание всех REST API-эндпоинтов приложения.

---

## Обзор

Базовый путь API: `/api`

Все эндпоинты принимают и возвращают данные в формате **JSON**. Кодировка — `UTF-8`.

### Аутентификация

Аутентификация не требуется — все эндпоинты открыты.

---

## Эндпоинты

### 1. Health Check

**`GET /api`**

Проверка работоспособности сервера.

#### Запрос

```
GET /api
```

#### Ответ `200 OK`

```json
{
  "message": "Hello, world!"
}
```

---

### 2. Получить все записи

**`GET /api/items`**

Возвращает список всех записей из базы данных, отсортированных по убыванию ID (новые сверху).

#### Запрос

```
GET /api/items
```

**Параметры:** отсутствуют

#### Ответ `200 OK`

```json
[
  {
    "id": 4,
    "name": "Деплой приложения",
    "description": "Развёртывание production-версии на сервере с настройкой окружения.",
    "createdAt": "2025-01-15T10:30:00.000Z",
    "updatedAt": "2025-01-15T10:30:00.000Z"
  },
  {
    "id": 3,
    "name": "Проектирование интерфейса",
    "description": "Создание адаптивного интерфейса с помощью Tailwind CSS и shadcn/ui.",
    "createdAt": "2025-01-15T10:30:00.000Z",
    "updatedAt": "2025-01-15T10:30:00.000Z"
  }
]
```

**Описание полей ответа:**

| Поле | Тип | Описание |
|------|-----|---------|
| `id` | `number` | Уникальный идентификатор записи |
| `name` | `string` | Название записи |
| `description` | `string \| null` | Описание записи (может быть `null`) |
| `createdAt` | `string` | Дата создания в формате ISO 8601 |
| `updatedAt` | `string` | Дата последнего обновления в формате ISO 8601 |

#### Ответ `500 Internal Server Error`

```json
{
  "error": "Не удалось загрузить записи"
}
```

---

### 3. Создать запись

**`POST /api/items`**

Создаёт новую запись в базе данных.

#### Запрос

```
POST /api/items
Content-Type: application/json
```

**Тело запроса:**

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|---------|
| `name` | `string` | **Да** | Название записи (не может быть пустым или содержать только пробелы) |
| `description` | `string` | Нет | Описание записи |

**Пример:**

```json
{
  "name": "Моя новая запись",
  "description": "Подробное описание записи"
}
```

#### Ответ `201 Created`

```json
{
  "id": 5,
  "name": "Моя новая запись",
  "description": "Подробное описание записи",
  "createdAt": "2025-01-15T12:00:00.000Z",
  "updatedAt": "2025-01-15T12:00:00.000Z"
}
```

#### Ответ `400 Bad Request`

Поле `name` отсутствует, пустое или содержит только пробелы:

```json
{
  "error": "Поле \"Название\" обязательно для заполнения"
}
```

#### Ответ `500 Internal Server Error`

```json
{
  "error": "Не удалось создать запись"
}
```

---

### 4. Обновить запись

**`PUT /api/items/[id]`**

Обновляет существующую запись по её ID.

#### Запрос

```
PUT /api/items/5
Content-Type: application/json
```

**Параметры URL:**

| Параметр | Тип | Описание |
|----------|-----|---------|
| `id` | `string` | ID записи для обновления (должен быть числом) |

**Тело запроса:**

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|---------|
| `name` | `string` | **Да** | Новое название записи |
| `description` | `string` | Нет | Новое описание записи |

**Пример:**

```json
{
  "name": "Обновлённое название",
  "description": "Новое описание"
}
```

#### Ответ `200 OK`

```json
{
  "id": 5,
  "name": "Обновлённое название",
  "description": "Новое описание",
  "createdAt": "2025-01-15T12:00:00.000Z",
  "updatedAt": "2025-01-15T13:30:00.000Z"
}
```

Обратите внимание: поле `updatedAt` автоматически обновляется при каждом изменении.

#### Ответ `400 Bad Request`

Некорректный ID или пустое название:

```json
{
  "error": "Некорректный ID записи"
}
```
```json
{
  "error": "Поле \"Название\" обязательно для заполнения"
}
```

#### Ответ `404 Not Found`

Запись с указанным ID не существует:

```json
{
  "error": "Запись не найдена"
}
```

#### Ответ `500 Internal Server Error`

```json
{
  "error": "Не удалось обновить запись"
}
```

---

### 5. Удалить запись

**`DELETE /api/items/[id]`**

Удаляет запись из базы данных по её ID.

#### Запрос

```
DELETE /api/items/5
```

**Параметры URL:**

| Параметр | Тип | Описание |
|----------|-----|---------|
| `id` | `string` | ID записи для удаления |

#### Ответ `200 OK`

```json
{
  "message": "Запись успешно удалена"
}
```

#### Ответ `400 Bad Request`

```json
{
  "error": "Некорректный ID записи"
}
```

#### Ответ `404 Not Found`

```json
{
  "error": "Запись не найдена"
}
```

#### Ответ `500 Internal Server Error`

```json
{
  "error": "Не удалось удалить запись"
}
```

---

### 6. Заполнить БД начальными данными

**`POST /api/items/seed`**

Добавляет 4 тестовые записи в базу данных, если таблица пуста.

#### Запрос

```
POST /api/items/seed
```

**Тело запроса:** отсутствует

#### Начальные данные

| Название | Описание |
|----------|---------|
| Изучение Next.js | Полный курс по фреймворку Next.js с App Router, Server Components и API Routes. |
| Работа с базой данных | Подключение SQLite через Prisma ORM. Выполнение CRUD-операций. |
| Проектирование интерфейса | Создание адаптивного интерфейса с помощью Tailwind CSS и shadcn/ui. |
| Деплой приложения | Развёртывание production-версии на сервере с настройкой окружения. |

#### Ответ `200 OK` (данные добавлены)

```json
{
  "message": "Добавлено 4 начальных записей в базу данных.",
  "count": 4
}
```

#### Ответ `200 OK` (данные уже существуют)

```json
{
  "message": "В базе уже есть 4 записей. Начальные данные не добавлены.",
  "count": 4
}
```

#### Ответ `500 Internal Server Error`

```json
{
  "error": "Не удалось добавить начальные данные"
}
```

## Коды ошибок

| Код | Значение | Когда возникает |
|-----|---------|----------------|
| `200` | OK | Успешное выполнение GET/PUT/DELETE или данные уже существуют |
| `201` | Created | Запись успешно создана |
| `400` | Bad Request | Неверный формат данных, отсутствует обязательное поле |
| `404` | Not Found | Запись с указанным ID не найдена |
| `500` | Internal Server Error | Ошибка на стороне сервера (БД, Prisma и т.д.) |

## Примеры использования с curl

```bash
# Получить все записи
curl /api/items

# Создать запись
curl -X POST /api/items \
  -H "Content-Type: application/json" \
  -d '{"name": "Тест", "description": "Тестовая запись"}'

# Обновить запись (ID = 1)
curl -X PUT /api/items/1 \
  -H "Content-Type: application/json" \
  -d '{"name": "Обновлено", "description": "Новое описание"}'

# Удалить запись (ID = 1)
curl -X DELETE /api/items/1

# Заполнить БД начальными данными
curl -X POST /api/items/seed
```

## Реализация

Все API-роуты реализованы как **Next.js Route Handlers** в директории `src/app/api/`:

```
src/app/api/
├── route.ts                 # GET /api → health check
└── items/
    ├── route.ts             # GET/POST /api/items
    ├── [id]/
    │   └── route.ts         # PUT/DELETE /api/items/[id]
    └── seed/
        └── route.ts         # POST /api/items/seed
```

Все роуты используют общий модуль `src/lib/db.ts` для доступа к Prisma Client:

```typescript
import { db } from '@/lib/db'

// Пример: получение всех записей
const items = await db.item.findMany({
  orderBy: { id: 'desc' },
})
```

Доступ к базе данных осуществляется **только на сервере** (server-side). Клиент взаимодействует с БД исключительно через HTTP API.
