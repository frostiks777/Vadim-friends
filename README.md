# Управление записями — База данных SQLite

**Курсовой проект** — веб-приложение для управления записями (CRUD) с подключением к базе данных SQLite через Prisma ORM. Построено на стеке **Next.js 16 + React 19 + TypeScript + Tailwind CSS 4 + shadcn/ui**.

> Часть данных выводится из кода на сайт, а другая часть — из базы данных.

## Обзор проекта

Данный проект представляет собой веб-приложение, разработанное в рамках курсовой работы. Основная задача — продемонстрировать работу с базой данных в связке с современным фронтенд-фреймворком.

### Ключевые принципы

| Принцип | Реализация |
|---------|-----------|
| **Фронтенд** | React 19, Tailwind CSS 4, shadcn/ui, Framer Motion |
| **Бэкенд** | Next.js 16 App Router, API Routes (Server-side) |
| **База данных** | SQLite через Prisma ORM |
| **Язык** | TypeScript 5 (строгая типизация) |
| **Данные** | Часть из кода (статика) + часть из БД (динамика) |

## Функциональные возможности

### Данные из кода (статическая часть)
- Отображение 6 карточек «Возможности проекта» с иконками, описаниями и тегами
- Отображение технологического стека (8 технологий) в виде сетки
- Статистика: количество возможностей, технологий, API-эндпоинтов

### Данные из базы данных (динамическая часть)
- **Создание (Create)** — форма добавления новой записи (название + описание)
- **Чтение (Read)** — список всех записей с сортировкой по убыванию ID
- **Обновление (Update)** — диалог редактирования существующей записи
- **Удаление (Delete)** — подтверждение удаления с AlertDialog
- **Начальное заполнение** — кнопка «Заполнить БД» для добавления 4 тестовых записей
- **Обновление списка** — кнопка ручного обновления данных

### UX-особенности
- Адаптивный дизайн (mobile-first)
- Состояния загрузки (skeleton-заглушки)
- Пустое состояние с подсказкой
- Toast-уведомления об успешных/ошибочных операциях
- Анимированный список с Framer Motion
- Hover-эффекты на карточках записей

## Установка и запуск

### Требования
- **Node.js** >= 18 или **Bun** >= 1.0
- **SQLite3** (входит в комплект с Prisma)

### Установка

```bash
# Клонирование репозитория
git clone https://github.com/iskorik303/-.git
cd -

# Установка зависимостей
bun install
```

### Настройка базы данных

```bash
# Применить схему к базе данных (создаёт таблицу Item)
bun run db:push

# Опционально: сгенерировать Prisma Client
bun run db:generate
```

Файл `.env` должен содержать:

```env
DATABASE_URL=file:./db/custom.db
```

### Запуск

```bash
# Режим разработки
bun run dev
# → http://localhost:3000

# Линтинг
bun run lint
```

### Production-сборка

```bash
bun run build
bun run start
```

## База данных

Подробная документация: [docs/DATABASE.md](docs/DATABASE.md)

### Модель данных

| Поле | Тип | Описание |
|------|-----|---------|
| `id` | `Int` (autoincrement) | Уникальный идентификатор |
| `name` | `String` (required) | Название записи |
| `description` | `String?` (optional) | Описание записи |
| `createdAt` | `DateTime` | Дата и время создания |
| `updatedAt` | `DateTime` | Дата и время последнего обновления |

### Скрипты для работы с БД

```bash
bun run db:push      # Применить схему к БД
bun run db:generate  # Сгенерировать Prisma Client
bun run db:migrate   # Создать и применить миграцию
bun run db:reset     # Сбросить базу данных
```

## API-эндпоинты

Подробная документация: [docs/API.md](docs/API.md)

### Обзор

| Метод | Путь | Описание | Статус |
|-------|-----|----------|--------|
| `GET` | `/api/items` | Получить все записи | 200 / 500 |
| `POST` | `/api/items` | Создать новую запись | 201 / 400 / 500 |
| `PUT` | `/api/items/[id]` | Обновить запись | 200 / 400 / 404 / 500 |
| `DELETE` | `/api/items/[id]` | Удалить запись | 200 / 400 / 404 / 500 |
| `POST` | `/api/items/seed` | Заполнить БД начальными данными | 200 / 500 |

### Пример запроса (POST)

```bash
curl -X POST /api/items \
  -H "Content-Type: application/json" \
  -d '{"name": "Моя запись", "description": "Описание записи"}'
```

### Пример ответа (GET)

```json
[
  {
    "id": 1,
    "name": "Изучение Next.js",
    "description": "Полный курс по фреймворку Next.js...",
    "createdAt": "2025-01-15T10:30:00.000Z",
    "updatedAt": "2025-01-15T10:30:00.000Z"
  }
]
```

## Данные из кода vs данные из БД

| Аспект | Данные из кода | Данные из БД |
|--------|---------------|-------------|
| **Источник** | Переменные в `page.tsx` (`STATIC_FEATURES`, `STATIC_TECH_STACK`) | Таблица `Item` в SQLite |
| **Тип данных** | Массивы объектов, определённые в исходном коде | Динамические записи из API |
| **Изменяемость** | Только через редактирование исходного кода | Через веб-интерфейс или API |
| **Отображение** | Карточки возможностей, технологический стек | Список записей с CRUD-операциями |
| **Секция страницы** | «Данные из кода» | «Данные из базы данных» |
| **Подключение** | Не требуется | Prisma ORM → SQLite |

## Структура страницы

Страница разделена на следующие визуальные секции:

1. **Header (шапка)** — фиксированная навигация с логотипом, названием и бейджами технологий
2. **Hero** — заголовок проекта с градиентным текстом и описанием
3. **Статистика** — 4 карточки с числовыми показателями
4. **Данные из кода** — сетка из 6 карточек возможностей + карточка технологического стека
5. **Разделитель** — визуальная граница между секциями
6. **Данные из БД** — форма добавления + список записей + справка по API
7. **Footer (подвал)** — информация о технологиях с иконками
8. **Диалоги** — модальное окно редактирования, подтверждение удаления
